# Contributing

感谢你为 `news-push` 做贡献。

## 开发要求

- 使用 Python 3.12
- 提交前先运行测试：

```bash
make test
```

- 如果修改了运行方式、环境变量或部署流程，请同步更新 [`./README.md`](./README.md)

## 提交流程

建议按下面流程提交：

1. 从最新 `main` 拉出分支
2. 完成修改并补充必要测试
3. 本地执行 `make test`
4. 提交 PR，并写清楚变更目的和验证方式

## Pull Request 要求

- 保持改动聚焦，不要把无关重构混进同一个 PR
- 如果改动涉及定时任务、环境变量或推送逻辑，请在 PR 描述中明确说明
- 如果修复 bug，最好说明复现方式和修复后的行为

## 问题反馈

如果你发现问题但暂时不准备提交代码，也欢迎直接提 issue，尽量附上：

- 运行环境
- 错误日志
- 复现步骤
- 预期行为
